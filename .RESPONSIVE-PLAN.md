# MeetNoo LP — Plan de fix responsive mobile (max-width:768px)

**Branche :** `feat/lp-mobile-quick-win`
**Date :** 2026-06-27
**Auteur :** Gabriel (analyse phase 1, exécution phase 2)

## 0. Constat structurel CRITIQUE (vaut pour TOUTES les pages)

Toutes les pages partagent une architecture **Figma-export "fixed-stage"** identique :

1. **Viewport figé** : `<meta name="viewport" content="width=1728" />` (ligne 30 de chaque page).
   → Sur mobile, le navigateur traite l'écran comme un canvas de 1728px.
2. **Stage scalé en JS** : un `<div id="stage" style="width:1728px">` enveloppe tout le contenu, et un `<script>` en bas applique `transform: scale(vw/1728)` pour rétrécir le tout.
   → Sur un iPhone 375px → scale 0.217 → tout devient illisible mais "proportionnel".
3. **Layout en position:absolute** : `.hero h1`, `.hero .lead`, `.hero .cta`, `.preview`, `.float-card`, `.stat-row`, etc. sont tous en `position:absolute` avec `left/top` en pixels durs.
4. **Containers en `width:1728px` dur** : `.hero`, `.sec`, `.platform`, `.transcription`, `.pricing`, `.testimonials`, `.blog`, `.footer`, `.contact-hero`, `.faq`, `.trust`, `.types`, `.bento`, etc.
5. **Grilles avec colonnes en px** : `grid-template-columns: 480px 1fr`, `repeat(3,370px)`, `560px 1fr`, etc.

### Stratégie globale (à appliquer à CHAQUE page)

Sur mobile, on doit **désactiver le scale du stage** ET réécrire le layout via `@media (max-width:768px)`. La règle de base à insérer dans chaque page (avant `</style>`) :

### Script à patcher (l.1864-1889)
Le script JS calcule `transform: scale()` sur `#stage`. Sur mobile, le CSS doit override avec `!important`. Le script s'exécute après et utilise `.style.transform` — donc **on doit aussi ajouter une garde JS** ou faire en sorte que le CSS gagne via `@media + !important`. **Recommandation phase 2** : ajouter à la fin du `fit()` :
```js
if (window.innerWidth <= 768) {
  stage.style.transform = 'none';
  stage.style.width = '100%';
  wrap.style.height = 'auto';
  navStage.style.transform = 'none';
}
```

## 9. Garde-fou JS (à ajouter à la fin du script `fit()` de CHAQUE page)

```js
// dans le scope de fit(), juste après le calcul du scale
if (window.innerWidth <= 768) {
  stage.style.transform = 'none';
  stage.style.width = '100%';
  wrap.style.height = 'auto';
  if (navStage) {
    navStage.style.transform = 'none';
    navStage.style.width = '100%';
    navStage.style.left = '0';
  }
  if (navFix) navFix.style.height = '64px';
  return;
}
```

Et idéalement remplacer la balise viewport sur les 5 pages :
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
```
…mais **uniquement à l'intérieur d'un `<script>` qui le réécrit conditionnellement** si on veut garder le scale desktop figé sur tablette horizontale. Alternative plus simple : laisser `width=1728` et confier 100% du travail au CSS @media + override JS ci-dessus.
